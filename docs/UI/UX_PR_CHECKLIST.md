# UX PR Checklist

이 체크리스트는 UI/UX 관련 PR의 최소 품질 게이트다.

## 1) 일관성
- [ ] 디자인 토큰(색/간격/반경/타입)을 재사용했다.
- [ ] 기존 패턴 컴포넌트를 우선 사용했고, 중복 구현을 만들지 않았다.
- [ ] 버튼/입력/탭/패널의 상태 표현이 기존 화면과 충돌하지 않는다.

## 2) 사용성
- [ ] 핵심 동작의 클릭 수/입력 수가 불필요하게 늘지 않았다.
- [ ] 작업 실패 시 복구 경로(재시도/리셋/되돌리기)가 있다.
- [ ] 로딩 중 사용자가 다음 상태를 예측할 수 있다.

## 3) 상태 UI
- [ ] Loading 상태가 존재하고 시각적으로 명확하다.
- [ ] Error 상태에 사용자 행동 가능한 안내가 있다.
- [ ] Empty 상태 문구가 맥락에 맞고 다음 행동을 제시한다.

## 4) 접근성
- [ ] 키보드-only로 핵심 흐름을 완료할 수 있다.
- [ ] 포커스 표시가 명확하고 끊기지 않는다.
- [ ] 버튼/입력/영역에 적절한 label/aria 속성이 있다.
- [ ] 텍스트 대비가 충분하고 색상만으로 의미를 전달하지 않는다.

## 5) 성능/안정성
- [ ] 불필요한 리렌더를 유발하지 않는다.
- [ ] 고빈도 업데이트(실시간 데이터)에서도 뷰 안정성이 유지된다.
- [ ] 빌드/타입체크가 통과한다.

## 6) 증빙
- [ ] 변경 전/후 스크린샷 또는 짧은 영상 첨부
- [ ] 영향을 받는 사용자 시나리오 명시
- [ ] 리스크/제한사항을 PR 설명에 기록
