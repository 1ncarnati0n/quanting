# UX Experiment Plan (Week 2)

## 목적
- UI/UX 개선 항목의 실제 효과를 정량으로 검증한다.
- 시행일 기준: 2026-03-02

## 실험 1: 상태 UI 통일 효과
### 가설
- Loading/Error/Empty 상태를 공통 패턴으로 통일하면 복구 성공률이 증가하고 재시도 시간이 감소한다.
### 대상
- ChartContainer, WatchlistSidebar
### 측정 지표
- 오류 발생 후 정상 복귀까지 소요 시간
- 사용자 재시도 성공률
- 상태 화면에서 이탈률
### 성공 기준
- 평균 복구 시간 20% 감소
- 재시도 성공률 15% 증가

## 실험 2: Settings 빠른 설정 허브 효과
### 가설
- 자주 쓰는 지표를 상단 허브에서 즉시 제어하면 설정 변경 완료 시간이 단축된다.
### 대상
- SettingsPanel > indicators 탭
### 측정 지표
- 지표 3개 변경 완료 시간
- 패널 내 스크롤 이동 횟수
- 잘못된 토글 후 원복 비율
### 성공 기준
- 평균 작업 시간 30% 단축
- 스크롤 이동 횟수 25% 감소

## 실험 3: 접근성 보강 효과
### 가설
- 포커스/ARIA 개선 시 키보드-only 완료율이 상승한다.
### 대상
- 사이드바 및 주요 인터랙션 영역
### 측정 지표
- 키보드-only 시나리오 완료율
- 포커스 상실/혼동 이벤트 수
### 성공 기준
- 완료율 90% 이상
- 포커스 관련 결함 50% 감소

## 수집 방법
1. 사전/사후 동일 시나리오를 수행한다.
2. 각 시나리오별 시간/실수/복구 횟수를 기록한다.
3. 정량 결과와 정성 피드백을 함께 보관한다.
