# UX 용어집 (v1)

## 목적
- 화면 전반의 문구/라벨을 통일해 인지 부하와 해석 오류를 줄인다.
- 신규 기능 추가 시 같은 의미의 용어가 중복 생성되는 것을 방지한다.

## 핵심 용어
- 관심종목: 사용자가 추적하는 종목 목록
- 종목: 화면에서 기본으로 사용하는 자산 식별 단위 표현
- 종목 코드: 종목을 직접 입력할 때 사용하는 코드(예: AAPL, BTCUSDT)
- 즐겨찾기: 관심종목 중 우선 추적 대상
- 현재 시장: 현재 선택된 종목의 시장(US/KR/코인/FX)
- 스크리너: 조건 기반 종목 탐색 기능
- 지표: 차트에 표시되는 분석 지표
- 오버레이 지표: 가격 차트 위에 겹쳐 표시되는 지표
- 오실레이터: 별도 보조 축에 표시되는 지표
- 신호 구간: 매수/매도 배경 구간 표시
- 기본 모드: 핵심 설정 중심의 단순화 화면
- 고급 모드: 전체 세부 설정 노출 화면

## 라벨 규칙
- 영어 혼합 라벨은 가능한 한 한글 우선으로 표기한다.
- 축약어는 업계 표준 약어(RSI, MACD, VWAP 등)만 유지한다.
- 동작 버튼은 명사보다 동사형으로 표기한다.
  - 예: `설정 내보내기`, `설정 불러오기`, `핵심 지표 켜기`
- 상태 표현은 일관된 표현을 사용한다.
  - 예: `없습니다`, `준비 중`, `갱신 중`

## 적용 예시
- `심볼 검색` → `종목 검색`
- `티커 직접 입력` → `종목 코드 직접 입력`
- `저장(Export)` → `설정 내보내기`
- `불러오기(Import)` → `설정 불러오기`
- `볼밴` → `볼린저`
- `신호구간` → `신호 구간`
- `UX 지표 Export` → `UX 지표 내보내기`

## 운영 규칙
- PR에서 UI 문구를 추가/변경하는 경우 이 문서를 먼저 확인한다.
- 신규 용어가 필요하면 기존 항목과 충돌 여부를 확인 후 항목을 추가한다.
