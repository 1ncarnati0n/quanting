# TradingView vs Quanting — 기능 갭 분석

> TradingView.com 기준 기능 비교 및 Quanting에서 추가 구현 가능한 항목 정리
>
> 참고: [TradingView Features](https://www.tradingview.com/features/) · [Chart Types](https://www.tradingview.com/support/solutions/43000703407-chart-types-available-on-tradingview/) · [Drawing Tools](https://www.tradingview.com/support/solutions/43000703396-drawing-tools-available-on-tradingview/)

## 범례

| 기호 | 의미 |
|------|------|
| ✅ | 이미 구현됨 |
| 🟢 | 프론트엔드만으로 구현 가능 (lightweight-charts API 또는 React) |
| 🟡 | 프론트 + 백엔드(Rust) 작업 필요 |
| 🔴 | 인프라/외부 서비스 필요 (대규모) |
| ⚪ | Quanting 성격에 맞지 않아 불필요 |

### 드로잉 도구 구현 현황

```
Phase 1: ✅ 수평선/추세선/피보나치 되돌림 (완료)
Phase 2: ✅ 평행 채널/직사각형/텍스트 주석 (완료)
Phase 3: ✅ 피치포크/갠/엘리엇/하모닉 (완료)
```

아키텍처:
- `src/stores/useDrawingStore.ts` — 드로잉 객체 상태 관리
- `src/components/DrawingCanvas.tsx` — Canvas 오버레이 레이어
- `src/components/DrawingToolbar.tsx` — 좌측 도구 패널
- 드로잉 데이터 localStorage 영속화
- 선택 강조 + Delete 키 삭제 + 개별/전체 삭제 UX
