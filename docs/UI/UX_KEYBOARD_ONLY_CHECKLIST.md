# UX Keyboard-Only Checklist

## 목적
- 마우스 없이 주요 사용자 흐름을 수행할 수 있는지 점검한다.
- 포커스 이동, 상태 인지, 실행/취소 경로가 키보드만으로 가능한지 검증한다.

## 점검 기준일
- 1차 점검일: 2026-03-02
- 2차 점검일: 2026-03-02
- 3차 점검일: 2026-03-02
- 점검 방식: 코드 기반 점검 + 단축키/핸들러 검증 + 모달 컨텍스트 충돌 검증 + 핵심 시나리오 계약 자동 점검

## 핵심 시나리오 체크리스트
1. 종목 검색 열기/닫기: `Ctrl/Cmd+K`로 열기, `Esc`로 닫기 가능
2. 종목 검색 결과 이동/선택: `↑/↓` 이동, `Enter` 선택 가능
3. 관심종목 리스트 선택: `Tab` 진입 후 `Enter/Space`로 종목 전환 가능
4. 관심종목 리스트 내 방향키 탐색: `↑/↓`, `Home/End`로 항목 포커스 이동 가능
5. 설정 탭 이동: `Tab`과 `Enter`로 `지표/레이아웃/화면` 전환 가능
6. 지표 토글 조작: `Tab` 이동 후 `Enter/Space`로 상태 변경 가능
7. 툴바 액션 조작: 차트/드로잉 툴바 버튼이 키보드 포커스 및 실행 가능
8. 모달 닫기 경로: 단축키 도움말/검색 모달이 `Esc`와 닫기 버튼으로 종료 가능
9. 포커스 표시: 버튼/입력/링크/role=button 요소에 visible focus ring 제공
10. 모달 열림 상태 보호: 모달/시트가 열린 상태에서 `F`, `R`, `?` 등 전역 단축키가 비활성화
11. 명령 허브 진입/실행: `Ctrl/Cmd+J`로 허브 열기, 검색 후 `Enter`로 액션 실행 가능
