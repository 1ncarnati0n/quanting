# Next Quarter UX Roadmap Draft

## 대상 기간
- 2026-05-25 ~ 2026-08-16 (12주)

## 목표
- 운영 텔레메트리 기반 KPI 실측 체계 완성
- 대규모 데이터(1,000+ 종목)에서도 체감 성능 유지
- 엔터프라이즈 보안/감사 관점의 UX 안정성 강화

## 스트림별 초안
### 1) UX Telemetry & Insights
- 로컬 이벤트를 세션/시나리오 단위로 집계하는 리포트 파이프라인 도입
- 오류 복구율, 단축키 사용률, 시나리오 완료시간 자동 산출
- KPI 대시보드 문서 주간 업데이트

### 2) Data-Dense Performance
- Watchlist/검색 결과 windowed virtualization 도입
- 차트 패널 동시 갱신 시 우선순위 렌더 스케줄링
- 장시간 세션 메모리 사용량 회귀 테스트 자동화

### 3) Accessibility 2.0
- 스크린리더 시나리오(NVDA/VoiceOver) 표본 점검 루틴 추가
- aria-live/명령 피드백 문구 표준화
- 고대비 테마 점검과 토큰 레벨 대비 감사 자동화

### 4) Enterprise Workflow UX
- 워크스페이스 템플릿/잠금(읽기전용) UX 도입
- 팀 협업용 설정 스냅샷 비교/복원 플로우 설계
- 대량 심볼 관리(태그/그룹/일괄작업) 기능 정교화

## 초기 마일스톤
1. M1 (2026-06-14): KPI 실측 파이프라인 v1 + 리포트 자동화
2. M2 (2026-07-12): 대용량 리스트 가상화 + 성능 회귀 게이트
3. M3 (2026-08-16): 접근성/엔터프라이즈 UX 운영 가이드 v2

## 리스크
- 텔레메트리 도입 범위가 과도하면 개발 속도를 저해할 수 있음
- 외부 데이터 변동성으로 성능 벤치마크 재현성이 흔들릴 수 있음

## 대응
- KPI 산출식과 계측 범위를 분기 초에 고정한다.
- 합성 시뮬레이션 + 런타임 표본 계측을 병행해 기준을 이중화한다.
